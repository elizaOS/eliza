<div id="prerequisites">
## Requisitos previos
</div>

Antes de instalar Eliza, asegúrate de tener el runtime requerido:

| Requisito | Versión | Cómo verificar | Instalar |
|-----------|---------|----------------|----------|
| **Node.js** | >= 22 | `node --version` | [nodejs.org](https://nodejs.org) o `nvm install 22` |

Para **compilar desde el código fuente** o desarrollo, también necesitas:

| Requisito | Versión | Cómo verificar | Instalar |
|-----------|---------|----------------|----------|
| **Bun** | última | `bun --version` | `curl -fsSL https://bun.sh/install \| bash` |

<Info>
Se requiere Node 22+ porque Eliza utiliza módulos ES nativos, el prefijo de protocolo `node:` y otras APIs modernas de Node. Bun solo es necesario para desarrollo y compilación desde el código fuente -- se usa como gestor de paquetes y ejecutor de scripts. El paquete npm publicado y los scripts de instalación se ejecutan solo con Node.js.
</Info>

También necesitarás una clave API de al menos un proveedor de modelos (Anthropic, OpenAI, OpenRouter, etc.) -- a menos que planees usar [Ollama](https://ollama.com) para inferencia completamente local, que no requiere clave API.

<div id="desktop-app-recommended">
## Aplicación de escritorio (Recomendada)
</div>

La aplicación de escritorio es la forma más fácil de comenzar. Empaqueta el runtime completo de Eliza, el panel web y el gateway en una aplicación nativa de Electrobun con actualizaciones automáticas e integración con la bandeja del sistema.

Descarga desde **[GitHub Releases](https://github.com/eliza-ai/eliza/releases/latest)**:

| Plataforma | Archivo | Notas |
|------------|---------|-------|
| macOS (Apple Silicon) | [`Eliza-arm64.dmg`](https://github.com/eliza-ai/eliza/releases/latest) | Macs M1/M2/M3/M4 |
| macOS (Intel) | [`Eliza-x64.dmg`](https://github.com/eliza-ai/eliza/releases/latest) | Macs anteriores a 2020 |
| Windows | [`Eliza-Setup.exe`](https://github.com/eliza-ai/eliza/releases/latest) | Windows 10+ |
| Linux | [`Eliza.AppImage`](https://github.com/eliza-ai/eliza/releases/latest) / [`.deb`](https://github.com/eliza-ai/eliza/releases/latest) | AppImage o paquete Debian |
| iOS | App Store (pronto) | iPhone y iPad |
| Android | [Google Play](https://play.google.com/store/apps/details?id=ai.eliza.app) / [APK](https://github.com/eliza-ai/eliza/releases/latest) | Android 10+ |

Todas las versiones están firmadas y notarizadas. Sin advertencias de Gatekeeper en macOS.

<div id="verify-download-integrity">
### Verificar la integridad de la descarga
</div>

```bash
cd ~/Downloads
curl -fsSLO https://github.com/eliza-ai/eliza/releases/latest/download/SHA256SUMS.txt
shasum -a 256 --check --ignore-missing SHA256SUMS.txt
```

<div id="cli-install">
## Instalación por CLI
</div>

<Tabs>
  <Tab title="macOS / Linux">
    El script de instalación descarga la última versión, coloca el binario en tu PATH y ejecuta la configuración inicial:

    ```bash
    curl -fsSL https://get.eliza.ai | bash
    eliza setup
    ```

    Luego inicia Eliza:

    ```bash
    eliza
    ```
  </Tab>
  <Tab title="Windows (PowerShell)">
    ```powershell
    irm https://get.eliza.ai/install.ps1 | iex
    ```

    Después de la instalación, abre una nueva ventana de terminal y ejecuta:

    ```powershell
    eliza setup
    eliza
    ```
  </Tab>
  <Tab title="Bun">
    Si tienes Bun instalado:

    ```bash
    bun install -g elizaai
    eliza setup
    eliza
    ```
  </Tab>
  <Tab title="npm">
    Si prefieres npm:

    ```bash
    npm install -g elizaai
    eliza setup
    eliza
    ```
  </Tab>
  <Tab title="Homebrew">
    ```bash
    brew tap eliza-ai/eliza
    brew install eliza          # CLI
    brew install --cask eliza   # Desktop app (macOS only)
    ```
  </Tab>
  <Tab title="Snap">
    ```bash
    sudo snap install eliza
    eliza setup
    ```

    Disponible en Ubuntu, Fedora, Manjaro y cualquier distribución con [snapd](https://snapcraft.io/docs/installing-snapd). Los paquetes Snap se actualizan automáticamente en segundo plano.

    Para las últimas versiones de desarrollo:
    ```bash
    sudo snap install eliza --edge
    ```
  </Tab>
  <Tab title="Flatpak">
    ```bash
    flatpak install flathub ai.eliza.Eliza
    flatpak run ai.eliza.Eliza
    ```

    O instala manualmente desde un [paquete de lanzamiento](https://github.com/eliza-ai/eliza/releases/latest):
    ```bash
    flatpak --user install eliza.flatpak
    ```
  </Tab>
  <Tab title="APT (Debian / Ubuntu)">
    ```bash
    # Add the repository
    curl -fsSL https://apt.eliza.ai/gpg.key | sudo gpg --dearmor -o /usr/share/keyrings/eliza.gpg
    echo "deb [signed-by=/usr/share/keyrings/eliza.gpg] https://apt.eliza.ai stable main" | \
      sudo tee /etc/apt/sources.list.d/eliza.list

    # Install
    sudo apt update && sudo apt install eliza
    ```

    Funciona en Debian 12+, Ubuntu 22.04+, Linux Mint 22+, Pop!_OS y otros derivados de Debian. Las actualizaciones llegan a través de `apt upgrade`.
  </Tab>
</Tabs>

<div id="build-from-source">
## Compilar desde el código fuente
</div>

Para colaboradores o cualquier persona que desee la última versión de desarrollo:

<Steps>
  <Step title="Clonar el repositorio">
    ```bash
    git clone https://github.com/eliza-ai/eliza.git
    cd eliza
    ```
  </Step>
  <Step title="Instalar dependencias">
    ```bash
    bun install
    ```

    El hook `preinstall` inicializa primero los submódulos de git; el `postinstall` posterior asegura que el directorio de habilidades exista, vincula el servidor del navegador y aplica parches a las dependencias.
  </Step>
  <Step title="Compilar todo">
    ```bash
    bun run build
    ```

    Esto compila el código fuente TypeScript con `tsdown`, escribe la información de compilación y construye la aplicación web en `apps/app`.

    <Info>
    `bun run build` ejecuta la compilación de producción vía Node (`scripts/run-production-build.mjs`).
    </Info>
  </Step>
  <Step title="Iniciar Eliza">
    ```bash
    bun run start
    ```

    Esto lanza el gateway y el servidor API. En la primera ejecución, se te guiará a través del flujo de incorporación interactivo.
  </Step>
</Steps>

<div id="development-mode">
## Modo de desarrollo
</div>

Eliza tiene varios modos de desarrollo dependiendo de en qué estés trabajando:

<CodeGroup>
```bash Full dev (runtime + UI hot reload)
bun run dev
```

```bash UI only (web dashboard hot reload)
bun run dev:ui
```

```bash Desktop app (Electrobun with live reload)
bun run dev:desktop
```

```bash Desktop app + Vite HMR
bun run dev:desktop:watch
```
</CodeGroup>

El script `dev` utiliza `scripts/dev-ui.mjs` para iniciar el runtime con recarga en caliente tanto para el backend como para la interfaz web. El script `dev:desktop` inicia la aplicación Electrobun con recarga en vivo.

<div id="environment-setup">
## Configuración del entorno
</div>

Eliza almacena todo el estado en `~/.eliza/` por defecto. Al iniciar por primera vez,
el cliente comienza eligiendo o conectándose a un servidor. Si estás creando un servidor
local, aún puedes preconfigurar los ajustes de arranque y las claves de proveedor con
variables de entorno o archivos de configuración.

<div id="api-keys">
### Claves API
</div>

Solo necesitas claves de proveedor locales cuando el servidor local debe usarlas. Si
planeas conectarte a un servidor LAN, remoto o Eliza Cloud existente, ese servidor
puede ya gestionar el enrutamiento y las credenciales del proveedor. Para el arranque
local, puedes establecer las claves del proveedor en `~/.eliza/eliza.json` bajo el
campo `env` o en `~/.eliza/.env`:

<CodeGroup>
```bash Anthropic (recommended)
export ANTHROPIC_API_KEY="<ANTHROPIC_API_KEY>"
```

```bash OpenAI
export OPENAI_API_KEY="<OPENAI_API_KEY>"
```

```bash OpenRouter
export OPENROUTER_API_KEY="<OPENROUTER_API_KEY>"
```

```bash Google Gemini
export GOOGLE_API_KEY="<GOOGLE_API_KEY>"
```

```bash Ollama (local, no key needed)
export OLLAMA_BASE_URL="http://localhost:11434"
```
</CodeGroup>

<div id="ports">
### Puertos
</div>

| Servicio | Puerto por defecto | Variable de entorno |
|----------|-------------------|---------------------|
| Dashboard + API | `2138` | `ELIZA_PORT` |
| Gateway (WebSocket) | `18789` | `ELIZA_GATEWAY_PORT` |
| WeChat Webhook | `18790` | `ELIZA_WECHAT_WEBHOOK_PORT` |
| Home Dashboard | `2142` | `ELIZA_HOME_PORT` |

En modo de desarrollo (`bun run dev`), la API se ejecuta en un puerto separado:

| Servicio | Puerto por defecto | Variable de entorno |
|----------|-------------------|---------------------|
| API + WebSocket (dev) | `31337` | `ELIZA_API_PORT` |
| Dashboard UI (dev) | `2138` | `ELIZA_PORT` |

```bash
# Example: custom ports
ELIZA_PORT=3000 ELIZA_GATEWAY_PORT=19000 eliza start
```

<div id="state-directory">
### Directorio de estado
</div>

| Variable | Por defecto | Descripción |
|----------|-------------|-------------|
| `ELIZA_STATE_DIR` | `~/.eliza` | Directorio raíz para todo el estado de Eliza |
| `ELIZA_CONFIG_PATH` | `~/.eliza/eliza.json` | Ruta explícita del archivo de configuración |

<div id="security-api-token">
## Seguridad: Token API
</div>

El servidor API se vincula a `127.0.0.1` (loopback) por defecto -- solo tu máquina local puede acceder a él. Si expones el servidor a la red (por ejemplo, `ELIZA_API_BIND=0.0.0.0` para despliegues en contenedores o en la nube), **debes** establecer un token de autenticación:

```bash
echo "ELIZA_API_TOKEN=$(openssl rand -hex 32)" >> ~/.eliza/.env
```

<Warning>
Sin un token en una dirección de enlace pública, cualquier persona que pueda alcanzar el servidor obtiene acceso completo al panel, los endpoints de agentes y las operaciones de billetera.
</Warning>

<div id="verify-installation">
## Verificar la instalación
</div>

Después de la instalación, confirma que todo está funcionando:

```bash
# Check version
eliza --version

# Show configured model providers
eliza models

# Bootstrap or validate local setup
eliza setup
```

El comando `models` lista los proveedores de modelos configurados y su estado para el
contexto del servidor local seleccionado. El comando `setup` arranca la configuración
local cuando es necesario y valida tu espacio de trabajo.

<div id="troubleshooting">
## Solución de problemas
</div>

<div id="node-version-too-old">
### Versión de Node demasiado antigua
</div>

```
Error: Eliza requires Node.js >= 22
```

Actualiza Node.js: `nvm install 22 && nvm use 22`, o descárgalo desde [nodejs.org](https://nodejs.org).

<div id="bun-not-found">
### Bun no encontrado
</div>

```
Error: bun: command not found
```

Instala Bun:
```bash
curl -fsSL https://bun.sh/install | bash
```

Luego reinicia tu terminal o ejecuta `source ~/.bashrc` / `source ~/.zshrc`.

<div id="port-already-in-use">
### Puerto en uso
</div>

```
Error: EADDRINUSE :::2138
```

Otro proceso está usando el puerto por defecto. Detenlo o usa un puerto diferente:
```bash
ELIZA_PORT=3000 eliza start
```

<div id="permission-denied-on-install-script">
### Permiso denegado en el script de instalación
</div>

```bash
chmod +x ~/.eliza/bin/eliza
```

<div id="pglite-database-corruption">
### PGLite / corrupción de base de datos
</div>

Si ves errores de base de datos al iniciar, Eliza intentará una recuperación automática. Si eso falla, puedes restablecer la base de datos:

```bash
rm -rf ~/.eliza/workspace/.eliza/.elizadb
eliza start
```

<Warning>
Eliminar el directorio de la base de datos borra todo el historial de conversaciones y la memoria. Usa esto como último recurso.
</Warning>

<div id="macos-gatekeeper-blocks-the-app">
### macOS Gatekeeper bloquea la aplicación
</div>

La aplicación de escritorio está firmada y notarizada. Si Gatekeeper aún la bloquea:

```bash
xattr -cr /Applications/Eliza.app
```

<div id="local-macos-desktop-builds-unsigned-no-apple-certs">
### Compilaciones locales de escritorio en macOS (sin firmar / sin certificados de Apple)
</div>

Los **lanzamientos** de GitHub están firmados y notarizados. Si **empaquetas la aplicación Electrobun localmente** sin credenciales de Developer ID, establece **`ELECTROBUN_SKIP_CODESIGN=1`** (o deja los secretos sin configurar en CI) para que la compilación no espere la firma de Apple.

En **macOS**, después de `node scripts/desktop-build.mjs package` (desde la raíz del repositorio), el constructor de escritorio ejecuta **`apps/app/electrobun/scripts/local-adhoc-sign-macos.ts`** sobre el **`.app`** producido para que el paquete esté **firmado ad hoc** para pruebas locales. También puedes ejecutarlo manualmente:

```bash
cd apps/app/electrobun && bun scripts/local-adhoc-sign-macos.ts /path/to/Eliza.app
```

La firma ad hoc **no** es notarización; Gatekeeper puede requerir **Clic derecho → Abrir** la primera vez. Contexto completo del pipeline: [Compilación y lanzamiento](/es/build-and-release.md).
