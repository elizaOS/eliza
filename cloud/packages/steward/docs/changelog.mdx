# Changelog

## @stwd/sdk

### 0.3.0 (2026-03-24)

- **Bearer token auth** — Agent JWTs now sent as `Authorization: Bearer` header (preferred over API key)
- **Solana signing** — `signSolanaTransaction()` method for Solana transaction signing
- **EIP-712 typed data** — `signTypedData()` for DEX approvals and permits
- **Batch agent creation** — `createWalletBatch()` with optional shared policies
- **Multi-wallet addresses** — `getAddresses()` returns both EVM and Solana addresses
- **RPC passthrough** — `rpcPassthrough()` for read-only chain queries
- **Token balances** — Query ERC-20 balances via the agents endpoint

### 0.2.0 (2026-03-10)

- **Policy management** — `getPolicies()` and `setPolicies()` methods
- **Transaction history** — `getHistory()` for audit trail
- **Balance queries** — `getBalance()` with chain ID support
- **Error handling** — `StewardApiError` with status and typed data

### 0.1.0 (2026-02-28)

- Initial release
- `createWallet()` — Agent creation with encrypted EVM wallet
- `signTransaction()` — EVM transaction signing with policy evaluation
- API key authentication

## Steward API

### v1.5.0 (2026-03-27)

- **Secret Vault** — Encrypted credential storage with CRUD API
- **Credential routes** — Route-based credential injection configuration
- **Agent deletion** — `DELETE /agents/:agentId` with cascade
- **Steward on all nodes** — `STEWARD_BIND_HOST=0.0.0.0` across 6 nodes
- Fixed cloud provisioning (correct prod DB, correct ports)
- Fixed bridge port (`DEFAULT_BRIDGE_PORT=2138`)
- Fixed discovery service (prod DB, milady.ai domain)

### v1.4.0 (2026-03-20)

- **Solana signing** — Full Solana transaction signing + broadcast
- **EIP-712 signing** — `eth_signTypedData_v4` support
- **Multi-chain wallets** — Agents get both EVM and Solana keypairs
- **RPC passthrough** — Proxy read-only RPC calls per agent
- **Platform API** — Cross-tenant management with platform key auth

### v1.3.0 (2026-03-14)

- **Batch agent creation** — Create multiple agents in one request
- **Agent tokens** — Generate scoped JWTs for agent authentication
- **Token balances** — ERC-20 balance queries
- **Key import** — Import existing private keys into the vault

### v1.2.0 (2026-03-07)

- **Webhook notifications** — Configurable webhooks for tx events
- **Approval queue** — Manual approval flow for high-value transactions
- **Policy engine** — 5 policy types (spending-limit, approved-addresses, auto-approve-threshold, time-window, rate-limit)

### v1.1.0 (2026-02-28)

- **Multi-tenant** — Tenant isolation with API key auth
- **SIWE auth** — Sign In With Ethereum for web dashboard
- **Transaction history** — Query agent transaction records

### v1.0.0 (2026-02-15)

- Initial release
- AES-256-GCM encrypted wallet storage
- EVM transaction signing with Argon2id key derivation
- Basic agent CRUD
- Hono + Bun API server
