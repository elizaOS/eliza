elizaOS spans a runtime, a plugin ecosystem, an app distribution layer, a hosted cloud, an operating system, and a model-training stack. The documentation is organized around eight tracks. Pick the one that describes what you're building.

## Tracks compose

These aren't silos. A typical project crosses several:

- A **Plugin** is consumed by an **Agent**, which is wrapped in an **Agent App**, which deploys to **Cloud**.
- A **Framework App** can target **elizaOS** as its host OS.
- **Training & Simulation** produces models that any of the above can load.

Start with the track closest to your output artifact and follow the links inward.
