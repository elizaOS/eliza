## LOW-CONFIDENCE / needs human decision

- **`orchestrator.py:899` versioning fallback `return "1.0.0"`.** Reading a
  missing `bundle/VERSION` and silently defaulting to `1.0.0` will
  produce a publish whose tag claims `eliza-1-<tier>-v1.0.0` regardless
  of the real upstream lineage. Consider failing-loud when VERSION is
  missing — operators who mean to publish 1.0.0 can write it.

- **`benchmarks/eliza1_gates.yaml`** marks 32 of 36 numeric gates
  `provisional: true` (every metric except barge-in, mobile-rss, and
  thermal). The publish gate still blocks on them. That's correct per
  AGENTS.md §6 ("provisional gates still gate"), but it means the
  first real publish will almost certainly trip a provisional that was
  set without measurement. Worth surfacing in the publish-runbook.

- **JSON Schema sibling
  `eliza-1.manifest.v1.json`** declares `publishedAt` as
  `format: "date-time"`, which per JSON Schema spec accepts offsets.
  Now in disagreement with both Zod (`Z`-only) and Python (`Z`-only).
  External consumers of the JSON Schema will accept manifests the
  runtime rejects. Consider tightening the JSON Schema with an explicit
  `pattern` mirroring the Python regex, or document the drift.
